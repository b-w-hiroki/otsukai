# 広告・お布施(応援)配置メモ

> おうちのおつかいアプリ向けの収益化・応援機能の配置案。実装前の検討メモ。

## 前提
- 家族向けのおつかいPWA(4タブ: お買い物 / ミッション / ストック / 設定)。
- **子どもも使う**(ミッション機能)ため、広告はファミリー向け/非パーソナライズが必須(COPPA等のポリシー注意)。
- 静的PWA + ログイン構成。AdSenseは独自ドメイン+審査が必要なため、まずは審査不要の**お布施リンク**から始め、広告は後追いが安全。
- クリーンな実用UIが強みなので、広告は最小限(リスト末尾1枠 + 完了時リワード程度)に絞るのを推奨。

## 広告(さりげなく、操作の邪魔をしない場所)

| 候補 | 場所 | 評価 |
|------|------|------|
| リスト末尾インフィード | お買い物タブ `#list-done` の後 / ストックタブ `#stock-ok-section` の後 | ◎ コンテンツの邪魔をせず自然。最下部で誤タップも少ない |
| 完了時リワード広告 | おつかい完了 / ミッション達成のモーダル | ○ 区切りが自然。「動画を見てごほうび+」など報酬型と相性良い |
| 設定タブ末尾 | ログアウトカードの下 | ○ 低頻度だが無難 |
| (避ける) | bottom-nav 周辺・FAB付近・各シート内の操作領域 | ✗ 誤タップ/操作妨害でUX悪化 |

## お布施(応援)

| 候補 | 場所 | 評価 |
|------|------|------|
| 設定タブに応援カード | 「☕ 開発者を応援する」カードを追加(Buy Me a Coffee / PayPal.me / Amazonギフト等のリンク) | ◎ 最も自然で実装も簡単。広告より先にこれだけ入れるのもアリ |
| 使い方モーダル末尾 | 既存の `btn-howto` モーダル内 | ○ 好意的な気持ちのタイミングで露出 |
| マイルストーン時 | 完了履歴で「100回達成!」等の節目に控えめ表示 | ○ 押し付けがましくない |

## おすすめの第一歩
1. 設定タブに「お布施(応援)」カードを追加。
2. お買い物リスト末尾に広告プレースホルダーを1枠。

実装時に必要な情報:
- お布施リンク先URL(Buy Me a Coffee 等)
- 広告ネットワーク(AdMob / AdSense / 手動バナー)

## i-mobile 実装メモ (2026-05-27)
- 配置箇所: お買い物タブ末尾 (`#ad-slot-requests`)。`#list-done` の再描画で消えないよう `#tab-requests` 直下の兄弟要素として配置。
- 表示制御: `.ad-content` に子要素があるときだけ `.ad-slot` に `has-ad` クラスを付けて表示(`initAdSlots`)。タグ未貼付時は枠自体が出ない。
- 規約対応: 「広告」ラベルを `--muted` カラーで明示(`.ad-label`)。
- 残タスク:
  - i-mobile 管理画面でスポット作成 → 発行された `<script src="..." async>` を `#ad-slot-requests` の `.ad-content` 内に貼り付け。
  - 家族向けアプリのため、管理画面で**アダルト等のカテゴリを除外設定**。
  - 効果を見て、② ストックタブ末尾 / ③ 設定タブ末尾 / ④ 完了履歴シート末尾 に追加検討。
